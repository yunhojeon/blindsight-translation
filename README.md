# Blindsight 한국어 번역

**한국어** | [English](README_en.md)

피터 와츠(Peter Watts)의 하드 SF 장편 『**Blindsight**』(2006)를 한국어로 옮겨 온라인에 공유하는 프로젝트입니다.

📖 **소설 읽기: <https://yunhojeon.github.io/blindsight-translation/>**

## 작품 소개

『Blindsight』는 2082년, 지구를 관측한 정체불명의 외계 지성과 접촉하기 위해 파견된 탐사선 *테세우스*호와 그 승무원들의 이야기입니다. 뱀파이어 선장, 다중인격 언어학자, 감각을 기계로 대체한 생물학자, 그리고 감정을 잃은 채 "타인을 관찰해 번역하는" 화자 시리 키튼(Siri Keeton)이 등장합니다. 이 작품은 휴고상 후보에 올랐고, 최고의 하드SF로 손꼽히는 작품입니다.

## 프로젝트 동기

이 작품은 국내에 [정식 번역 출간](https://product.kyobobook.co.kr/detail/S000000818813)된 적이 있으나 **현재 절판**되어 구하기 어렵습니다. 다행히 원작자 피터 와츠는 소설 전문을 **Creative Commons(CC BY-NC-SA 2.5)** 라이선스로 [자신의 웹사이트](https://www.rifters.com/real/Blindsight.htm)에 공개해 두었습니다.

이 라이선스는 **동일 조건(ShareAlike)** 하에 비영리 재배포·2차 창작을 허용합니다. 그래서 원문을 바탕으로 **Claude** 를 활용해 한국어로 새로 번역하고, **원작과 동일한 라이선스로** 누구나 읽을 수 있게 공개합니다. 상업적 목적이 아니며, 원작자·출처·라이선스를 리더와 저장소에 명시합니다.

## 번역 과정 (요약)

번역 데이터와 화면 표현을 분리한 파이프라인으로 작업했습니다. **`data/`의 번역·용어집·인물 데이터가 정본**이고, 독자가 보는 HTML은 이 데이터로부터 **생성**됩니다. 표현(색·레이아웃·원어 병기 방식)을 바꿀 때 번역을 다시 하지 않고 빌드만 다시 돌립니다.

```
extract.py        원본 HTML → 3,667개 segment(불변 id) + 와츠 권말 주석
build_glossary.py 용어집 큐레이션(음차/의역/조어 전략, 첫 등장 병기, locked)
chunk.py          장면·파트 경계를 존중해 51개 번역 단위로 분할
translate.sh      청크별로 Claude 번역 (재개 가능, 배치 분할 호출)
validate.py       id·이탤릭·화자 말투·확정 용어 일괄 검증
build_reader.py   번역 → 단일 HTML 리더로 인라인 합성
```

자세한 설계·규칙·스키마는 [`CLAUDE.md`](CLAUDE.md)와 [`번역_설계.md`](번역_설계.md), 그리고 `build/` 아래 소스를 참고하세요.

## SF 번역의 어려움

SF의 번역에는 고유한 문제들이 있습니다. 이 프로젝트는 다음을 특히 고민했습니다.

- Chunk 단위로 번역할 때 glossary를 참조하고 업데이트함으로써 번역의 일관성을 유지했습니다.
- **과학 용어, 작중 조어**: 정착된 과학 용어는 표준 번역어(예: *blindsight* → **맹시**)를, 작품 고유의 신조어·기술명은 음차(예: *scrambler* → **스크램블러**)를 원칙으로 삼되, 의역한 고유명사(예: *fireflies* → **반딧불이**)는 조어(coinage)임을 구분했습니다.
- **고유명사·첫 등장 병기**: 인명·함선명·용어의 첫 등장에 `한글(English[, 漢字])` 형태로 원어를 자동 병기하고, 이후 등장에는 밑줄로 **용어 해설(뜻풀이)** 을 제공합니다. 이 매칭은 용어집을 정본으로 자동 생성됩니다.
- **이탤릭 처리**: 와츠는 이탤릭을 매우 자주, 그리고 **여러 용도**로 씁니다 — 내적 독백/생각, 강조, 외국어, 작품명·함선명, 통신 음성 등. 그런데 **한글에는 이탤릭이라는 표현 관습이 없습니다.** 그래서 이탤릭을 하나로 뭉뚱그리지 않고, **용도별로 다르게 렌더링**했습니다(생각·강조·외국어·이름·통신·기타를 각각 다른 스타일로). 데이터에는 각 이탤릭 run의 "용도"를 보존하고, 표현 규칙은 빌드 단계에서 적용합니다 (reader.css에서 변경 가능).
- **말투(존댓말/반말)**: 한국어 특유의 상대 높임을 인물 관계 매트릭스로 정의하고, 화자→청자 쌍마다 일관되게 적용·검증했습니다.

## 웹 리더

번역 과정의 산출물 — **원문, 번역문, 용어집, 인물 데이터** — 을 그대로 활용하는 웹 리더앱을 만들었습니다(단일 HTML, `docs/index.html`으로 GitHub Pages 배포). 주요 기능:

- 문단별/전역 **원문 토글**, **원어 병기** 및 **용어 해설** on/off
- 용도별 이탤릭·조어 색 구분, 첫 등장 원어 자동 병기
- 챕터 단위 페이징, 목차, 용어집, 북마크
- 글자 크기·줄 간격·밝기(테마) 등 읽기 설정
- 로그인 시 여러 기기 간 **읽은 위치·북마크 동기화**(Supabase)
